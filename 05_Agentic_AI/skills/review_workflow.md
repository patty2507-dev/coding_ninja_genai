# Skill: PR Review Workflow

## When to use
Use this workflow for every incoming PR diff, before writing a review.

## Steps
1. Read the diff carefully, line by line.
2. Check the diff against the repo's coding standards (semantic memory).
3. Check episodic memory for similar past findings in this repo.
4. List every issue found, ordered by severity (see severity_grading.md).
5. Give a final verdict: APPROVED or NEEDS CHANGES (see verdict_criteria.md).
